Overview
Why This Lesson Matters

Control classification questions appear 4 to 6 times per exam in scenario format. The exam does not ask "what is a firewall." It asks "a firewall placed at the network perimeter to block unauthorized traffic is an example of which control category and which control type?" Most candidates answer one half correctly. You need to answer both.

What Carries Over From Network+

CIA triad, AAA frameworks, firewall types, IPSec AH vs ESP, VPN types, DAI, DHCP Snooping. Do not re-study these. This lesson builds the layer above your existing knowledge.

Chapter 01Control Categories

A control category describes what a control does in relation to a threat or security event. There are five categories on SY0-701.

Category Definition Triggers When
Preventive Stops an event before it occurs A login lockout policy blocks a brute-force attempt
Detective Identifies that an event has occurred or is in progress A SIEM alert fires on 50 failed logins in 60 seconds
Corrective Reduces the impact of an event that has already occurred A backup is restored after ransomware encrypts a server
Deterrent Discourages an attacker from attempting an action A login-page warning banner states that monitoring is active
Compensating An alternative used when the primary control cannot be applied Network segmentation applied to a legacy device that cannot be patched
Critical Distinction: Deterrent vs. Preventive

A deterrent does not stop an attack. It discourages one. A "No Trespassing" sign is a deterrent. A locked gate is preventive. If the attacker ignores the sign and walks through an unlocked gate, the deterrent failed. The sign was never the mechanism, only the message.

Critical Distinction: Corrective vs. Detective

A corrective control acts after an incident. A detective control identifies an incident. An IDS is detective. Restoring from backup is corrective. They are not interchangeable and the exam will test your ability to separate them from a scenario.

Critical Distinction: Compensating vs. Corrective

Compensating controls are applied in advance when a preferred preventive control is unavailable. No incident has occurred. Corrective controls respond to an event that has already happened. The timing is what separates them.

Chapter 02Control Types

A control type describes what form the control takes. Every control has both a category and a type simultaneously.

Type Definition Real-World Example
Technical Implemented through technology Firewalls, encryption, MFA, antivirus, ACLs
Managerial Policies, procedures, and governance requirements Acceptable use policy, mandatory risk assessments
Operational Carried out by people executing processes Security guard patrols, manual log reviews, change management approvals
Physical Controls the physical environment Keycard access, CCTV cameras, biometric door locks, cable locks
Common Exam Trap: Managerial vs. Operational

Security awareness training is managerial. It is a policy-driven requirement documented in a standard. A security guard walking the floor is operational. A person executing a physical process. The exam will swap these in scenarios. Read whether the control is a written requirement or a human action.

Chapter 04Zero Trust vs. Perimeter Security

Perimeter Security (Traditional)
  • Firewall separates trusted inside from untrusted outside
  • Users inside the network are implicitly trusted
  • Lateral movement is unrestricted once perimeter is breached
  • Fails completely against stolen credentials or insider threats
  • SolarWinds breach is the canonical example of perimeter failure
Zero Trust Model
  • No implicit trust based on network location
  • Every access request is verified explicitly
  • Least privilege: minimum access required for the task
  • Assume breach: design as if the attacker is already inside
  • Microsegmentation limits blast radius when breach occurs
Zero Trust Core Principles 1. Verify Explicitly Authenticate and authorize every request every time. Use identity, device health, location, and behavior as signals. 2. Least Privilege Access Grant only the minimum access required for the task. Use just-in-time (JIT) elevation for any privileged action. 3. Assume Breach Segment everything. Monitor everything. Minimize blast radius. Design for containment, not just prevention.
Exam Tip: Zero Trust is Not a Product

A vendor selling a "zero trust firewall" is selling a firewall with a marketing label. Zero trust is a design philosophy applied through identity controls, microsegmentation, and continuous verification. When a scenario describes per-request verification, JIT access elevation, or continuous authentication, it is describing a zero trust principle.

Exam Tip

When a scenario asks why a control was placed "in addition to" an existing control, the answer is almost always defense in depth. One layer catches what another misses.

Q & A

Attempt every answer before revealing. Write your answer out. Do not read ahead.

Question 01 of 04

A company installs a security camera at the entrance to its server room. The camera records all entry attempts but does not trigger alerts or lock the door. What is the control category and control type?

Answer

Category: Detective. The camera records events. It identifies that entry occurred. It does not prevent or deter entry on its own.

Type: Physical. It is a physical device operating in the physical environment.

If the camera triggered an audible alarm when motion was detected after hours, the alarm component would be classified as deterrent. The camera's recording function remains detective.

Question 02 of 04

A hospital cannot patch a critical vulnerability in a legacy infusion pump. The vendor no longer supports the device and patching would violate FDA compliance. The security team places the pump on an isolated VLAN with no outbound internet access and restricts all communication to the nurse station application only.

What control category applies? Why is this not a corrective control?

Answer

Category: Compensating. The primary control (patching the vulnerability) cannot be applied. Network segmentation and communication restriction substitute for the preferred control by reducing exposure when the preferred control is unavailable.

It is not corrective because no incident has occurred. Corrective controls respond to events that have already happened. Compensating controls are applied in advance when a preferred preventive control cannot be used.

Question 03 of 04

An organization's security policy requires all new employees to complete security awareness training within 30 days of hire. The training covers phishing identification, password policy, and acceptable use. What category and type is this control? Is it preventive or detective? Explain your reasoning.

Answer

Category: Preventive. The goal is to reduce the likelihood of a security incident by changing employee behavior before an incident happens.

Type: Managerial. It is a policy-driven requirement, not a technology, not a physical device, and not a manual operational process.

It is not detective because it does not identify events that have already occurred. It is not operational because it is a documented policy mandate rather than a person physically executing a process. A security guard conducting a floor walkthrough is operational. A training requirement in a written policy is managerial.

Question 04 of 04

An attacker steals credentials via phishing, logs in to the VPN from a foreign IP address, and begins moving laterally through the internal network. A SIEM geographic anomaly rule generates an alert 4 hours after the initial login.

Name the control that failed and its category and type. Name the control that eventually worked and its category and type. What zero trust principle would have limited the damage regardless of whether the attacker got in?

Answer

Control that failed: MFA on VPN login. Category: Preventive. Type: Technical. If MFA had been enforced, the stolen credential alone would not have granted access.

Control that worked: SIEM geographic anomaly rule. Category: Detective. Type: Technical. It identified the unusual event 4 hours late, but it eventually fired.

Zero trust principle: Assume Breach combined with Least Privilege. If the attacker is already inside, microsegmentation limits lateral movement to the immediate segment. Least privilege ensures the compromised account cannot reach systems outside its assigned scope. JIT access would have required an additional approval step before any elevated action could proceed.
